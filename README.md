### Update an existing project's features

#### Use Cases
##### Add a feature
##### Update a feature
##### Remove a feature
##### Resolve a bug

#### Steps
##### Discovery
When we determine wether we need to analyse & document the project.
We have to analyse and document the project if the project has not been created using Toddinator or the project has changed since the last time toddinator was used with this project.
When we then proceed to analyse, outline & document the project as a whole and each of it's file individually.

##### Documentation indexing
When we store the generted documentation for easy later retrieval, using embeddings & pinecone.

##### Documentation Retrieval
When, using embeddings, we identify & retrieve the appropriate documentation & identify the relevant files.

#### Metamorphosis
When we generate the new files' content, the existing files' updated content, and the list of files to be deleted.

##### Materialisation
When the files are the created, updated & deleted.

##### Iteration
When, using & feeding back the build, run & test steps outputs into the system, we iterate & converge towards a satisfactory output.

##### Finalisation
To be determined
