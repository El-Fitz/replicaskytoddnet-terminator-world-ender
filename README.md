# ReplicaSkyToddNet Terminator World-Ender (Toddinator, for short)


⚠️ An early (archived) attempt at *bootstraping* something *like* an agentic coding harness in January / March 2023, using itself (first raw OpenAI curls, then progressively switching to the tool itself as it gained in capabilities), before the release of Function Calling ([June 2023](https://openai.com/index/function-calling-and-other-api-updates/)), JSON mode ([November 2023](https://klu.ai/blog/openai-devday-2023) I believe, but not sure), and Structured Outputs ([August 2024](https://openai.com/index/introducing-structured-outputs-in-the-api/)).

It turned out to be an uphill battle as GPT-3.5 and model capqbilities proved insufficient and unreliable.

Yes, there's a leaked OpenAI, but it has been revoked.

### Update an existing project's features

#### Use Cases
##### Add a feature
##### Update a feature
##### Remove a feature
##### Resolve a bug

#### Steps
##### Discovery
When we determine wether we need to analyse & document the project.
We have to analyse and document the project if the project has not been created using Toddinator or the project has changed since the last time toddinator was used with this project.
When we then proceed to analyse, outline & document the project as a whole and each of it's file individually.

##### Documentation indexing
When we store the generted documentation for easy later retrieval, using embeddings & pinecone.

##### Documentation Retrieval
When, using embeddings, we identify & retrieve the appropriate documentation & identify the relevant files.

#### Metamorphosis
When we generate the new files' content, the existing files' updated content, and the list of files to be deleted.

##### Materialisation
When the files are the created, updated & deleted.
